Privacy Policy

Effective Date: April 6, 2026

1. Introduction

This "Sandbox" is a private, personal integration designed to automate the transfer of Oura Ring health data into a private Pigment workspace for personal analytics. This policy outlines the specific data accessed and how it is handled.

2. Data Collection (Oura API Scopes)

This application requests access to the following data categories from the Oura API v2:

  • Identity: Email and Personal profile information.
  • Daily Summaries: Daily activity, readiness, and sleep scores.
  • Biometrics: Heart rate (resting and continuous), SpO2 (blood oxygen), and Heart Health metrics.
  • Wellness & Stress: Stress levels, resilience scores, and tagged "Moments" or Sessions.
  • Activities: Recorded workouts and active heart rate data.
  • Device Info: Ring configuration and hardware status.

3. Data Usage & Destination

The data is used solely for Francesca's personal tracking. The workflow is as follows:

  • Data is pulled from the Oura Cloud API using OAuth2 credentials.
  • Data is transformed into a CSV format compatible with Pigment.
  • Data is pushed via API to a private Pigment workspace (Transaction Lists).

4. Data Storage and Security

  • No Third-Party Sharing: Data is never sold, shared, or used for marketing.
  • Minimal Retention: No health data is stored permanently on the server or script runner. It exists only in transit between Oura and Pigment.
  • Secret Management: API keys and OAuth tokens are stored as encrypted secrets and are not hard-coded or publicly accessible.

5. Controlling Your Data

As the sole user, you may revoke this application's access at any time through your Oura Cloud account settings. Revoking access will immediately stop the data sync to Pigment.
